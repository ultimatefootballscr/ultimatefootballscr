## 🛠️ Troubleshooting Common Issues

❌ **Script GUI Does Not Appear After Execution**
The most common cause is an outdated execution environment that doesn't support the latest Roblox API hooks. Make sure your tool is updated to its latest version and that you attached to the correct Roblox process.
**Solution:** Update your execution environment, restart Roblox, and re-inject the **Ultimate Football Script** fresh.

❌ **Auto Catch Triggers Too Early or Too Late**
This typically happens when the `timing_offset` value in your config doesn't match your current ping to the server. Higher latency requires a larger offset value.
**Solution:** Open `config.lua` and adjust `timing_offset` from `0.02` to `0.04` or `0.05` depending on your connection, then reload the **Best Ultimate Football Script** build.

❌ **FPS Drops After Extended Sessions**
Older versions had a known memory leak in the ESP module. If you're on anything below `v2.8.1`, this is likely the cause.
**Solution:** Update to the latest version and confirm `espEnabled` resets properly between matches. Restart your execution environment every 60 minutes during marathon sessions.

❌ **Speed Modifier Gets You Rubber-Banding**
Setting the `speedMultiplier` too high causes the server to reject your position updates, resulting in teleporting or snapping back to previous locations.
**Solution:** Keep `speedMultiplier` at or below `1.20` for the **Ultimate Football Script No Key Roblox** build. Values above that threshold are detectable on most server configurations.

❌ **Keybinds Not Responding In-Game**
Roblox occasionally captures key inputs before the script can register them, especially when chat or a menu is focused.
**Solution:** Click somewhere on the game viewport to ensure Roblox has focus, then try your keybind again. If the issue persists, remap the conflicting key in your **Auto Catch Ultimate Football Script** config file.
